🖨️ Radikant-Printer-C

A library for printing monochrome images and text from c on the Nimmbot label printer over UART. 

Introduction

The Nimmbot label printer is a cost effective battery operated thermal label printer, it can connect over UART and BLE to a client that can print text and images. This library is a reverse engineerd UART driver for this model. Technically Radikant BLE should be able to use the same protocol defined in this driver but hasnt been tested.  In a later stage i would like to make this library a more generic Printer driver for Thermal Label printers by extending it with ESC/POS and ZPL. This is an early protoype that needs some polishing in a later stage and decoupling of components such as image proccesing into a seperate library. Currently works only with 24 bit .bpm and .ppm files.

Pipeline

Memory Architecture

The driver uses a tiered image structure, transitioning from high-fidelity 24-bit RGB to optimized 1-bit monochrome buffers. This ensures that every byte of memory is used purposefully before being sent to the printer.

Resampling

Unlike simple "nearest-neighbor" scaling that leaves images looking blocky, this driver implements Lanczos Resampling and Bilinear Interpolation.
Using the Lanczos kernel allows the driver to downscale complex logos or text while maintaining crisp edges, which is vital for the 203 DPI (8 dots/mm) resolution of thermal printers.

Thermal printers only have black and white pixels. This code features a custom bit-packing engine that converts standard image data into a compact 1-bit-per-pixel format, specifically handling the bit-offset math required to feed the Niimbot hardware.

luminance conversion

  • Step 1 Luminance

Y = 0.299R + 0.587G + 0.114B to transform colors into grayscale. 

  • Step 2 Threshold

If Gray is < 128  = Black bit (1)
If Gray is > 128  = White bit (0)
This accounts for human eye sensitivit.

The Binarization Pipeline
RGB INPUT
GREEN (0, 255, 0)
→
WEIGHTED LUMA
GRAY: 150
G * 0.587
→
BIT OUTPUT
BIT: 0 (WHITE)
150 > 128

The Result: Because the green pixel is "perceptually bright" (150), it stays white. Without weighting, it would calculate to 85, cross the threshold, and print as an unwanted black smudge.
