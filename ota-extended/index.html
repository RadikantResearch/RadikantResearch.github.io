This library is only compatible with flash size sector of 4096 Bytes. To be compatible with ESP32 flash encryption, the entry size is 32 bytes. For integer types, an entry holds one key-value pair. For strings and blobs, an entry holds part of key-value pair (more on that in the entry structure description). Strings and Blobs will span multiple entries.

  • Header 32x1 bytes

  • Bitmap 32x1 bytes

  • Entries 32x126 bytes

2. Header

2.2 Content

The header is a 32 byte array and contains some simple information such as state, sequency number, version and CRC32 of this information. There are also 19 unused bytes which are set to 0xFF Strangly nvs_partition_gen.py always set the Page State to NVS_STATE_FULL = 0xFFFFFFFC
